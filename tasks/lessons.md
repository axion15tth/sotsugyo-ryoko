# Lessons

## Project

- セッション開始時に、作業ディレクトリの中身と `git` 管理ルートが一致しているかを先に確認する。
- `zsh` で `gh api -f source[branch]=...` のような引数を渡すときは、`'source[branch]=main'` のようにクォートする。
- 中断後に再開するときは、未完コマンドの副作用を `git status` と実行プロセスで確認してから次に進む。
- 旅行しおりの公開内容は、最新のユーザー指定を優先し、実名よりも公開向きの呼び名へ差し替える。
- ユーザーが文体を指定したら、事実情報だけでなく見出しや行程説明のトーンまで反映してから公開する。
- デザイン変更では情報構造を壊さず、まず CSS 主導で世界観を寄せると反映が速く安全。
- レイアウト修正では、削除指定された文言やセクションを残さず、見出し順や情報密度も同時に調整する。
- スナップスクロールを入れるときは、長い1セクションを無理に維持せず、内容をページ単位に分割した方が自然に見える。
- 横スワイプUIでは、横スナップだけでなくデスクトップ向けの補助操作も入れると確認しやすい。
- 横スワイプでアンカー移動を使うときは、リンククリックを補足して `scrollIntoView` で横方向に揃える。
- ユーザーが具体的な人物紹介文やページ構成を指定したら、既存の創作文や装飾よりその指定を優先して置き換える。
- 日程変更では、概要、イベント詳細、場所ガイドの整合を同時に更新し、古い行き先や時間帯を残さない。
- 役立つリンクを増やすときは専用ページを作り、個別ページにも戻り導線付きで点在させると使いやすい。
- ネタページを足すときは本筋の旅程を邪魔しないよう後半へ逃がし、日程側からは任意で飛べる形に留める。
- HTML に外部 URL やコードを直書きするときは、`&` や `<` をエスケープして表示崩れや解釈崩れを防ぐ。
- タイトルのような見た目要求は、文言だけでなく配置ルールそのものを専用マークアップで表現した方が再修正に強い。
- 外部サイト埋め込みでは、iframe だけに頼らず、地図埋め込みと Wikipedia API 要約カードを併用すると実表示が安定する。
- 無駄情報を増量するときは、1ページを長文化するだけでなく、関連する別ページへ分割した方が漫画的なページ送りとも相性がよい。
- ユーザーが「無駄情報を真面目な顔で載せたい」と言ったら、見出しや本文からメタな自虐を抜き、資料としては本気のトーンに寄せる。
- 年表を求められたら、印象論ではなく一次資料で拾えた具体的な年月日を優先し、日付粒度の異なる項目は混ぜ方を明示する。
- 非上場子会社の業績分析では、単独 PL が公開されていない可能性を前提に、親会社のセグメント情報を用いて推定であることを明記する。
- 表紙のタイポ調整は、文言だけでなく位置・太さ・サイズの3軸を同時に触ると、ユーザーの見た目要求に素直に追従しやすい。
- 目次やメンバー一覧のラベル修正では、表紙・一覧・関連導線の表記ゆれを残さない。
- 表紙デザインは、凝った構成よりもユーザーの可読性要求が優先されたら、専用演出をためらわず外して単純な1行タイトルへ戻す。
- 表紙の右上表記は absolute の見た目に頼るより、上段の明示的な左右レイアウトにした方がズレにくい。
- 複数行タイトルを絶対に崩したくないときは、grid 任せにせず `display:block` と明示改行で固定する。
- 表紙タイトルを目立たせたいときは、行分割だけでは足りず、線幅・太さ・マーカー色まで他ページ見出しと揃えると意図が伝わりやすい。
- マーカー装飾を文字列幅に合わせたいときは、タイトル行を `width:100%` のブロックにせず `inline-block` にする。
- 横スワイプ構成で広告ページを差し込むときは、目次には出さず、数ページおきに独立セクションとして挟むと漫画アプリ風のテンポを作りやすい。
- 演出として強い要素を足すときは、グロテスクに寄せず、文面と装飾で空気だけ変える方が公開ページでは扱いやすい。
- 数学補足を厳密化するときは、元の比喩を残さず、前提・係数導出・収束の順で段階的に分けると読みやすい。
- 広告をリアルに見せたいときは、抽象ネタだけで終わらせず、カテゴリチップ、CTA、注意書き、短い便益説明のようなUI記号を入れると一気に広告らしくなる。
- 広告を実在しそうに見せたいときは、アプリ名、カテゴリ、DL数や評価、機能説明を具体化し、冗談語や比喩を本文から外す。
- 外部作品へのリンク演出を入れるときは、参照先の実在URLを確認した上で、公開ページでは短いボタン導線に圧縮した方が扱いやすい。
- 本文として読ませたいページでは、共通カードUIをそのまま当てず、ページ種別ごとに padding・border・background を外す方がしおりらしい。
- 隠しリンク演出を作るときは、可視テキストを出さずに `aria-label` だけを持つリンク要素へまとめると見た目を壊さず実装できる。
- 縦に短い広告ページで初期表示が下にずれて見えるときは、`align-content: center` を避けて上寄せにすると安定する。
- 本文を手書き寄りに保ちたいときは、広告コンテナだけでサンセリフに切り替え、本文系の `p` や `li` は本文フォントを明示しておくと崩れにくい。
- 目次に現在地を出すときは、目次リンク対象だけでなく全ページの現在位置を見て、表紙や広告ページでも自然なラベルに落とす。
- 漫画風UIで時刻を目立たせたいときは、単なる色変更より丸囲みと少し傾いたスタンプ風の方がしおりらしさが出る。
- 紙ものの小物演出は HTML を増やさず、背景レイヤーに細い金属色グラデーションを重ねると全ページへ安全に展開しやすい。
- 装飾の解像度は上げすぎず、ユーザーが求める記号的な見た目に留める。ホチキスなら普通は銀の縦棒2本で十分。
