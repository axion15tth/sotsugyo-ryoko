Appendix

東京駅の歴史

東京駅丸の内駅舎の主要変遷

東京駅の沿革を押さえると、なぜ丸の内駅舎が都市の象徴であり続けているのかが見えやすい。とくに 1908 年着工から 1914 年開業、1945 年戦災、2007 年以降の保存・復原工事という節目は、駅の見え方そのものを決めている。

下の年表は、JR 東日本の技術レビュー、東京ステーションシティの学習コンテンツ、JR 東日本の公式資料で確認できた日付を優先して並べている。戦前・戦後・新幹線・保存復原という4つの層に分けて読むと、東京駅が単なる乗換拠点ではなく都市インフラと景観資産を兼ねていることが分かる。

集合地点の地図へ戻る

沿革年表

  • 1908年3月25日 東京駅丸の内駅舎が着工。
  • 1914年12月20日 「東京駅」として営業開始。
  • 1923年9月1日 関東大震災発生。丸の内駅舎は大きな被害を受けず使用継続。
  • 1945年5月25日 23時過ぎ 空襲により丸の内駅舎が炎上。
  • 1945年5月26日 7時過ぎ 火災が鎮火。
  • 1945年5月 東京駅女子高等学校が駅構内に開設。
  • 1947年3月 旧3階建から2階建への復旧工事が完了。
  • 1964年7月 東海道新幹線東京駅が完成。
  • 1964年10月1日 東海道新幹線が開業。
  • 2003年4月18日 文化審議会が東京駅丸の内駅舎の重要文化財指定を答申。
  • 2003年5月30日 東京駅丸の内駅舎が国指定重要文化財となる。
  • 2007年5月30日 東京駅丸の内駅舎保存・復原工事の起工式。
  • 2012年3月末 保存・復原工事の外観がほぼ姿を現す。
  • 2012年10月1日 JR EAST Travel Service Center と東京ステーションギャラリーが開業。
  • 2012年10月3日 東京ステーションホテルが開業。
  • 2024年12月20日 東京駅が開業110周年を迎える。

Appendix

東京駅史から集合設計

丸の内駅舎の象徴性と運用導線を分けて考える

東京駅の歴史を追うと、丸の内駅舎は「都市の顔」としての役割を強く持つ一方、利用者の移動は常にホーム接続や改札配置といった運用導線に支えられてきたことが分かる。観光と集合を同時に満たそうとすると、象徴空間と実運用空間を混同しやすい。

今回の集合点を新幹線南のりかえ口前に置くのは、その切り分けをした結果としてかなり合理的である。JR 利用のまま改札内で合流でき、東海道新幹線ホームへの移動距離も短く、集合後の行動がそのまま乗車に接続する。

東京駅史を踏まえると、集合設計では「写真映え」より「流線の単純さ」を優先する方が、巨大ターミナルの設計思想とも整合する。東京駅を景観資産として味わう時間と、列車に確実に接続する時間は、分けて扱うのがよい。

集合最適化問題:
入力: 人数 = 3, 駅 = 巨大, 乗車列車 = 東海道新幹線
目的関数: 迷子コスト + 遅刻コスト + 余計な歩行コスト を最小化
制約: 改札内集合、視認性、乗換のしやすさ
解の候補: 新幹線南のりかえ口前

運営会社の概要

京都タワーホテルアネックスの運営主体

京都タワーホテルアネックスの運営主体は 京阪ホテルズ＆リゾーツ株式会社 である。公式の利用規約では同社が京都タワーホテルアネックス内施設を運営する当事者として記載されており、同社の会社概要では 1959 年 6 月 1 日設立、資本金 1 億円、京阪ホールディングスの100%子会社とされている。

会社概要上の事業は「ホテル・レストラン業等」で、京都タワーホテルアネックスだけでなく、京都タワーホテル、京都センチュリーホテル、琵琶湖ホテルなど、観光・宿泊需要の強い拠点群を束ねている。京都駅前立地のアネックスは、この中でも短期滞在客との相性がよいポジションにある。

運営主体を把握しておくと、単館ではなくグループ運営の一部として見る視点が持てる。駅前立地、京都タワー周辺資産、同一エリア内の複数ブランド運営は、需要の取り込み方と価格戦略を考える上でかなり分かりやすい構図になっている。

京都駅周辺 or 京都タワーの論理式

自然言語の「or」を包摂的に読む

2日目 11:20 の「京都駅周辺 or 京都タワーをぶらつく」は、形式的には `K OR T` と表せる。ここで `K` は京都駅周辺を歩く、`T` は京都タワーに寄る、を意味する。

実運用では、この `or` を排他的 `XOR` と読む必要はない。包摂的 `OR` として読めば `K = 1, T = 1` も有効であり、駅周辺で土産を見てから京都タワーへ移るという行動列もきれいに表現できる。

時間バッファ `B` と体力 `E` を入れるなら、意思決定は `Decision = (K OR T) AND (B > 0) AND (E > 0)` のような形で扱える。予定表を命題化すると、曖昧な文言をどこまで柔らかく運用してよいかが見えやすい。

  • `K = 1, T = 0`: 駅ナカと土産で終える。
  • `K = 0, T = 1`: タワーへ一直線に行く。
  • `K = 1, T = 1`: 駅周辺もタワーも回収する。
  • `K = 0, T = 0`: 休憩を優先して待機する。

Appendix

行程のフーリエ級数展開

区分的な行程を調和成分で眺める

周期 `2π` の関数 `f(x)` は、おおまかに言えば `a0/2 + Σ(an cos nx + bn sin nx)` の形に分解できる。波を細かい正弦波と余弦波の和で見る発想であり、旅程の複雑さを周波数成分に分けることはできないが、気分だけは理系になる。

行程表は本来離散的だが、区分的な活動量関数として見れば、集合・移動・観光・食事といったイベントの密度を時間軸上に置き直せる。そこにフーリエ級数を重ねると、行程をなめらかな成分の和として理解する視点が得られる。

たとえば 8:00 集合、9:00 出発、11:15 京都着、19:30 夕食といった時刻列を、活動量の立ち上がりと減衰の並びとしてみると、低周波成分は「1日の大きな流れ」を、高周波成分は「細かな移動や切り替え」を表す、と解釈できる。

function squareWaveApprox(x, terms = 10) {
  let sum = 0;
  for (let k = 1; k <= terms; k += 1) {
    const n = 2 * k - 1;
    sum += Math.sin(n * x) / n;
  }
  return (4 / Math.PI) * sum;
}

Appendix

係数導出と別言語コード

直交性と近似誤差を確認する

フーリエ係数は三角関数の直交性を使って取り出す。`an = (1/π)∫f(x)cos(nx)dx`、`bn = (1/π)∫f(x)sin(nx)dx` と書けるので、行程を区分的な関数に見立てれば、活動量の主成分をどの周波数帯が担っているかを調べることもできる。

矩形波近似で現れるギブズ現象は、不連続な切り替え点でオーバーシュートが残る現象である。旅程でいえば、集合から乗車、観光から夕食といったモード切替の周辺に、なめらかな近似だけでは吸収しきれない変化が残ることに対応づけられる。

JavaScript と Python の両方で近似を書いておくと、同じ数式を複数の実装系でなぞれる。資料としては過剰だが、定式化の一貫性は確認しやすい。

import math

def square_wave_approx(x, terms=10):
    total = 0.0
    for k in range(terms):
        n = 2 * k + 1
        total += math.sin(n * x) / n
    return 4.0 * total / math.pi

xs = [i * 0.1 for i in range(-30, 31)]
ys = [square_wave_approx(x, terms=20) for x in xs]
